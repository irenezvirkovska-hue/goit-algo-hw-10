## Висновки

### Завдання 1

Було реалізовано два алгоритми для видачі решти: жадібний алгоритм (`find_coins_greedy`) та алгоритм динамічного програмування (`find_min_coins`).

Для суми **113** обидва алгоритми дали однаковий результат:

```python
{50: 2, 10: 1, 2: 1, 1: 1}
```

Під час тестування на великій сумі (**100000**) жадібний алгоритм показав значно кращу продуктивність:

- Жадібний алгоритм: **0.002280 секунд за 1000 запусків**
- Динамічне програмування: **0.349888 секунд за 10 запусків**

Жадібний алгоритм працює швидше, оскільки просто проходить по списку монет і на кожному кроці вибирає найбільший можливий номінал.

Алгоритм динамічного програмування працює повільніше, тому що перевіряє всі можливі варіанти для кожної проміжної суми, але гарантує знаходження оптимального рішення.

Для цього набору монет жадібний алгоритм є найкращим вибором, оскільки він швидкий і дає правильний результат.

### Завдання 2

У цьому завданні значення визначеного інтеграла функції `f(x) = x²` на проміжку від `0` до `2` було обчислено методом Монте-Карло з використанням **100 000 випадкових точок** та перевірено за допомогою функції `quad` з бібліотеки SciPy.

Результати показали, що метод Монте-Карло дає наближене значення, дуже близьке до точного результату.

Метод Монте-Карло є ефективним для наближених обчислень, особливо для складних задач, де аналітичне рішення знайти важко. Його точність залежить від кількості згенерованих точок: чим їх більше, тим точніший результат.

Функція `quad` дає точніший результат і добре підходить для перевірки правильності обчислень.
